Mais notre calcul donne unlocked_until = 2025-11-10 pour le dernier paiement de juillet 2026 — c'est clairement faux.

Le bug identifié

Notre formule cumule les jours depuis registration_date + 30j :

unlocked_until = registration_date + 30j + SUM(nb_jours_tous_paiements)

registration_date = 2024-08-16
+30j = 2024-09-15
+ SUM(nb_jours) jusqu'au 02/07/2026

Calculons manuellement :

daily_rate = 19 500 / 30 = 650 XOF/jour
Total payé = 273 800 XOF
nb_jours_total = 273 800 / 650 = 421 jours
unlocked_until = 2024-09-15 + 421j = 2025-11-10 ✅

C'est mathématiquement correct ! Le client a payé 421 jours depuis le début mais la période de contrat est de 765 500 / 19 500 × 30 = 1 177 jours.

Pourquoi CLD=0 alors ?

Parce que SURGE ne fonctionne pas comme UPYA. Dans SURGE, chaque paiement recharge le compteur depuis maintenant, pas depuis le début du contrat. C'est un système de balance, pas d'accumulation linéaire.

La logique correcte est :

unlocked_until(après paiement à date D) = 
    MAX(unlocked_until_précédent, D) + nb_jours_achetés

Si le client est en retard et paie, le kit se rallume depuis aujourd'hui, pas depuis la date de début.

On doit reconstruire unlocked_until de façon incrémentale par paiement :

Ce qu'on a appris

La formule correcte est incrémentale :

unlocked = MAX(unlocked_précédent, date_paiement) + nb_jours

Pas une somme depuis le début — chaque paiement repart du MAX entre la date actuelle et la date de déverrouillage précédente.

Maintenant on reconstruit en SQL

C'est une window function récursive — on va utiliser une CTE récursive ou generate_series pour reconstruire unlocked_until pour chaque contrat à chaque date de snapshot.

La bonne approche en SQL :

sql
WITH ranked_tx AS (
    SELECT
        contract_number,
        transaction_date,
        nb_jours,
        registration_date,
        ROW_NUMBER() OVER (
            PARTITION BY contract_number 
            ORDER BY transaction_date ASC
        ) as rn
    FROM gold.unified_transactions t
    JOIN gold.unified_contracts uc USING (contract_number)
)

Mais PostgreSQL ne supporte pas les CTEs récursives avec agrégats fenêtrés facilement.

La meilleure approche ici est Python — on fait le calcul en mémoire contrat par contrat et on insère le résultat.

bash
python -c "
import sys; sys.path.append('.')
from dotenv import load_dotenv
from database.db_client import get_db_connection
from datetime import date, timedelta
import psycopg2.extras
load_dotenv()
conn = get_db_connection()
cur = conn.cursor()

# Créer table unlocked_until corrigée
cur.execute('''
    CREATE TABLE IF NOT EXISTS gold.unlocked_until_history (
        contract_number TEXT,
        transaction_date DATE,
        unlocked_until  DATE,
        source          TEXT,
        PRIMARY KEY (contract_number, transaction_date)
    )
''')
cur.execute('TRUNCATE gold.unlocked_until_history')
conn.commit()

# Charger toutes les transactions et registration_dates
cur.execute('''
    SELECT
        t.contract_number,
        t.transaction_date::date,
        t.nb_jours,
        t.source,
        uc.registration_date::date
    FROM gold.unified_transactions t
    JOIN gold.unified_contracts uc ON uc.contract_number = t.contract_number
    WHERE t.payment_class IN ('upfront', 'recharge')
    ORDER BY t.contract_number, t.transaction_date ASC
''')
rows = cur.fetchall()
print(f'Transactions chargées : {len(rows):,}')

# Grouper par contrat
from collections import defaultdict
contracts = defaultdict(list)
for contract, tx_date, nb_jours, source, reg_date in rows:
    contracts[contract].append((tx_date, float(nb_jours), source, reg_date))

# Calculer unlocked_until pour chaque transaction
results = []
for contract, txs in contracts.items():
    reg_date = txs[0][3]
    unlocked = reg_date + timedelta(days=30)
    source = txs[0][2]
    for tx_date, nb_jours, src, _ in txs:
        new_start = max(unlocked, tx_date)
        unlocked = new_start + timedelta(days=nb_jours)
        results.append((contract, tx_date, unlocked, src))

print(f'Résultats calculés : {len(results):,}')

# Insérer en batch
psycopg2.extras.execute_values(
    cur,
    '''INSERT INTO gold.unlocked_until_history 
       VALUES %s ON CONFLICT DO NOTHING''',
    results,
    page_size=5000
)
conn.commit()
print(f'Insérés : {cur.rowcount:,}')
cur.close(); conn.close()
"
